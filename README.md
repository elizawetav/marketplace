# Анализ product/market fit и ключевых метрик роста маркетплейса товаров из Бразилии

#### Цель и задачи проекта
Провести комплексный анализ пользовательского поведения и бизнес-метрик маркетплейса для оценки удержания клиентов, определения наличия product/market fit и поиска точек роста выручки без ухудшения клиентского опыта.

Задачи проекта:
- Провести когортный анализ и оценить месячный retention пользователей в оформление заказа
- Проанализировать динамику заказов и пользовательской активности
- Оценить наличие product/market fit на основе поведенческих и продуктовых метрик
- Определить ключевые метрики, влияющие на рост прибыли маркетплейса
- Приоритизировать продуктовые гипотезы с помощью фреймворка ICE
- Определить метрики эффективности выбранной гипотезы
- Сформулировать выводы и подготовить аналитический отчет с рекомендациями для продуктовой команды

#### Методология и инструменты
- исследовательский анализ данных (EDA)
- Python (pandas, numpy, matplotlib, seaborn)
- когортный анализ
- анализ пользовательского поведения
- расчет продуктовых и бизнес-метрик
- приоритизация гипотез по фреймворку ICE

#### Источники данных
Использовались внутренние данные маркетплейса:
- olist_customers_dataset.csv — данные клиентов;
- olist_orders_dataset.csv — информация о заказах и их статусах;
- olist_order_items_dataset.csv — данные о товарах внутри заказов.

#### Результаты проекта
- Медианный retention 1-го месяца для всех когорт составил 0.35%, практически все когорты показывают резкое падение удержания после первой покупки. Анализ retention показал отсутствие устойчивого пользовательского поведения и низкую вероятность повторной покупки.
- На основе низкого уровня retention был сделан вывод об отсутствии product/market fit. Рост выручки обеспечивается преимущественно привлечением новых пользователей, а не возвратом существующих. 
- Выбраны 5 основных метрик для максимизации прибыли: выручка; количество уникальных пользователей, получивших заказ в этом месяце; CR; Customer Retention, ARPU
- Рассмотрены 3 продуктовых гипотезы и с помощью методики ICE выбрана одна - про исправление проблем в системе процессинга заказов. Определены метрики, на которые гипотеза должна повлиять: количество доставленных заказов, выручка от доставленных заказов, коэффицент конверсии в доставку заказа. 
- Сформулирована проблема продукта, общие выводы по итогу исследования и рекомендации для улучшения ситуации
